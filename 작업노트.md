## 변경 이력 (커밋 단위)

### 1) 게임 메커니즘 8종 개편 — `f3f0f71`
1. **산책**: 1회 호감도 +15% → **+5%**, **5분 쿨타임** 추가 (하루 3회 제한 유지)
2. **가혹 날씨**: 호감도 하락 속도 **20% 빠르게** (−0.1 → −0.12 /초)
3. **잠 방해**: **2분 내 3회 이상** 깨우면 호감도 **−20%p**, **🔔 깨우기 버튼** 신설
4. **버튼**: 크기 축소 + **글자 가로 정렬**(`white-space:nowrap`) + 줄바꿈 래핑
5. **🖱️ 이동/선택 모드 토글**: 땅=이동, 물건=선택(선택 링)
6. **청소**: 전체청소 버튼 제거 → 똥·쓰레기 **개별 클릭** 청소
7. **장난감·옷**: 설치/착용 시 호감도 **+3~10% 랜덤** 상승
8. **마당 확장**: 반지름 5.8 → **11** (약 2배) + **펫을 따라가는 팔로우 캠**

### 2) "화면(3D)이 사라지는" 문제 방어 — `e2c0873`
UI(상태바·패널)는 남고 3D만 사라지는 증상에 대한 견고화:
- **카메라 NaN/이상값 가드** — 비정상 값이면 안전 시점으로 즉시 복구
- **WebGL 컨텍스트 분실 자동복구** — `webglcontextlost/restored` 처리
- **탭 복귀 시 시간차 flush** — 백그라운드 후 첫 프레임 폭주 방지
- **렌더 루프 try/catch + 항상 렌더** — 한 프레임 오류가 화면을 영구히 죽이지 않게
- **GPU 리소스 dispose** — 똥/쓰레기/아이템/옷/펫 제거 시 정리(메모리 누수 차단; 장시간 플레이 시 컨텍스트 분실 근본 원인)

### 3) 모바일/터치 환경 대응 — `ee9bcd3`
PC는 정상, 모바일에서 발생한 문제들 수정:
1. **쓰레기/똥 선택 안 됨** → 탭하면 **선택**(청록 링) + 하단 **🧹 치우기 버튼** 노출 후 청소 (2단계)
2. **화면 이동 어려움** → 상호작용을 **손 뗄 때(탭)** 로 변경(드래그는 카메라 조작에 양보). **이동/선택 OFF 시 한 손가락 드래그로 화면 PAN**, 팔로우 캠도 해제
3. **로그인 시 4등분/깨짐** → `setPixelRatio→setSize` 순서 교정(+상한 2), 캔버스 `touch-action:none`, 뷰포트 **줌 금지**(`maximum-scale=1, user-scalable=no`), 회전 리사이즈 대응
4. **stale 캐시로 빈 화면 반복** → **세이브 버전(=2)** 으로 호환 안 되는 옛 캐시 자동 폐기, 손상 캐시 try/catch 비우기, `sanitizeState`로 잘못된 좌표/배열 필터링, add* 가드, **`_headers` no-store**로 항상 최신 에셋

### 4) 모바일 화면용 디버그 콘솔 — `ccf5444`
- 주소에 **`?debug=1`** 이 있을 때만 화면용 콘솔(eruda) 로드
- 사용: 태블릿 크롬에서 `3d-pet-dog.pages.dev/?debug=1` 접속 → 구석의 콘솔 아이콘 탭 → Console 탭에서 오류 확인
- 일반 접속(파라미터 없음)에는 영향 없음

## 모바일 조작법 (현재 기준)
- **🖱️ 이동/선택 ON**: 땅 탭 = 펫 이동 / 똥·쓰레기 탭 = 선택 후 [치우기] / 물건 탭 = 선택. 카메라는 펫을 따라감.
- **🖱️ 이동/선택 OFF**: 한 손가락 드래그 = 화면 자유 이동(PAN), 펫은 건드리지 않음.

## 확인이 필요한/미해결 사항
- **모바일 4등분·빈 화면**: 가장 유력한 원인들을 모두 막았으나, 실제 모바일 GPU/터치는 데스크톱 프리뷰로 재현이 불가. **`?debug=1` 콘솔 로그**로 재발 시 원인 확정 필요.
- 캐시 관련: 이번 배포부터 `_headers` no-store가 적용되므로, **딱 한 번** 브라우저 캐시 삭제 후 접속하면 그 이후부터는 자동으로 최신 유지.
